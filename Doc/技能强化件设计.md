## 二、技能强化件属性数据对象定义

### 2.1 数值口径说明

- 本文档中的每级属性值，均表示该强化件升到该等级后的最终属性值，不是“本级相对上一级的增量”。
- 字段名统一使用英文，便于程序配置、表结构定义与客户端读取。
- 字段名后缀含义统一如下：
  - `Pct`：百分比型属性，按乘法生效。
  - `Add`：固定值属性，按加法生效。
- 百分比型属性的口径：如暴击率、暴击伤害、命中率、闪避、处决阈值、吸血等，均表示对对应属性做百分比乘算修正。
- 固定值型属性的口径：如攻击力、技能冷却、弹幕数量、护盾值、领域伤害、召唤数量等，均表示对对应属性做固定值加减。
- `技能冷却` 虽然是 `N` 型字段，但业务上通常用于“减少冷却时间”；正式配置时可通过正负号表达增减方向。

### 2.2 技能强化件数据对象（skillEnhancementItemConfig）

```json
{
  "itemId": 0,
  "level": 1,
  "attackAdd": 0,
  "critRatePct": 0,
  "critDamagePct": 0,
  "hitRatePct": 0,
  "skillCooldownAdd": 0,
  "projectileCountAdd": 0,
  "dodgePct": 0,
  "executeThresholdPct": 0,
  "lifeStealPct": 0,
  "shieldValueAdd": 0,
  "auraDamageAdd": 0,
  "auraDurationAdd": 0,
  "auraRadiusAdd": 0,
  "buffDurationAdd": 0,
  "summonCountAdd": 0,
  "summonAttackAdd": 0,
  "summonDurationAdd": 0
}
```

说明：
- 该对象作为 JSON 配置使用。
- 每一条配置对应某个强化件道具在某个等级下的最终属性数据。
- 强化件品质不放在属性对象中，避免同一个技能在各等级属性配置中重复填写。
- 若某强化件不涉及某类属性，该字段填 `0`。

### 2.3 字段定义说明

| 字段名 | 中文含义 | 类型 | 生效口径 | 说明 |
|------|------|------|------|------|
| `itemId` | 道具ID | int | 标识字段 | 技能强化件道具唯一ID |
| `level` | 等级 | int | 标识字段 | 当前强化件等级，建议范围1~30 |
| `attackAdd` | 攻击力N | float | 加法 | 对攻击力做固定值加成 |
| `critRatePct` | 暴击率% | float | 乘法 | 对暴击率做百分比修正 |
| `critDamagePct` | 暴击伤害% | float | 乘法 | 对暴击伤害做百分比修正 |
| `hitRatePct` | 命中率% | float | 乘法 | 对命中率做百分比修正 |
| `skillCooldownAdd` | 技能冷却N | float | 加法 | 对技能冷却时间做固定值调整，通常负值表示减冷却 |
| `projectileCountAdd` | 弹幕数量N | int | 加法 | 对技能弹幕/子弹数量做固定值增加 |
| `dodgePct` | 闪避% | float | 乘法 | 对闪避率做百分比修正 |
| `executeThresholdPct` | 处决阈值% | float | 乘法 | 对处决触发血量阈值做百分比修正 |
| `lifeStealPct` | 吸血% | float | 乘法 | 对吸血比例做百分比修正 |
| `shieldValueAdd` | 护盾值N | float | 加法 | 对护盾数值做固定值增加 |
| `auraDamageAdd` | 领域伤害N | float | 加法 | 对领域类效果的伤害做固定值增加 |
| `auraDurationAdd` | 领域持续时间N | float | 加法 | 对领域持续时间做固定值增加 |
| `auraRadiusAdd` | 领域半径N | float | 加法 | 对领域作用半径做固定值增加 |
| `buffDurationAdd` | Buff持续时间N | float | 加法 | 对 Buff 持续时间做固定值增加 |
| `summonCountAdd` | 召唤数量N | int | 加法 | 对召唤物数量做固定值增加 |
| `summonAttackAdd` | 召唤物攻击力N | float | 加法 | 对召唤物攻击力做固定值增加 |
| `summonDurationAdd` | 召唤物持续时间N | float | 加法 | 对召唤物持续时间做固定值增加 |

## 三、技能强化件升级数据对象定义

### 3.1 升级数据对象（skillEnhancementUpgradeConfig）

```json
{
  "itemId": 0,
  "quality": 1,
  "upgradeCosts": [
    {
      "level": 1,
      "fragmentItemId": 0,
      "costItemCount": 1
    }
  ]
}
```

说明：
- 该对象作为 JSON 配置使用。
- 每一条配置对应某个强化件的一组升级消耗数据。
- 升级对象与属性对象拆分定义，避免把升级消耗和当前等级属性混在同一条配置里。
- `quality` 放在升级对象顶层，用于描述该技能强化件本身的品质信息，避免在每级属性配置里重复填写。
- 读取升级配置时只依赖 `itemId` 与当前 `level`；`quality` 不作为升级消耗的独立查询条件。
- `upgradeCosts` 中每一条数据表示当前等级升到下一级的消耗。
- `level` 表示当前等级；例如 `level=1` 表示从1级升到2级的消耗。
- `fragmentItemId` 表示升下一级消耗的强化碎片ID。
- `costItemCount` 表示升下一级消耗物品数量，对应上方每级消耗表。

### 3.2 升级字段定义说明

| 字段名 | 中文含义 | 类型 | 说明 |
|------|------|------|------|
| `itemId` | 道具ID | int | 对应技能强化件道具ID |
| `quality` | 品质 | int | 物品品质，建议使用 `1`=普通、`2`=稀有、`3`=史诗、`4`=传说、`5`=金色 |
| `upgradeCosts` | 升级消耗列表 | array | 存放该强化件各等级升下一级所需消耗 |
| `level` | 当前等级 | int | 表示当前强化件等级；该等级升到下一级时使用对应消耗 |
| `fragmentItemId` | 强化碎片ID | int | 升下一级所需强化件碎片ID |
| `costItemCount` | 消耗道具数量 | int | 升下一级所需碎片数量 |

## 四、技能强化件存储与读取口径

### 4.1 存储口径

- 技能强化件按背包系统中的可堆叠物品对象存储。
- 背包中的技能强化件记录复用 `stackItemRecord` 结构，至少包含：`itemId`、`itemType`、`level`、`count`、`hasExpireAt`、`expireAt`。
- 技能强化件的当前等级直接存放在可堆叠物品对象的 `level` 字段中；未升级时默认 `level=1`。
- 同一背包下同一 `itemId`、同一 `level`、同一过期批次的技能强化件按一条可堆叠记录聚合存储。

### 4.2 读取规则

- 先从背包中读取技能强化件对应的可堆叠物品对象。
- 再根据 `itemId + level` 读取当前等级属性配置。
- 再根据 `itemId` 对应的升级配置与当前 `level` 读取当前等级升下一级的升级消耗配置。
- 物品读取只与 `itemId + level` 有关；`quality` 从对应升级配置中带出，用于描述物品本身，不参与升级消耗计算。
- 对外返回时，将上述三部分组合为同一个技能强化件详情对象。

## 五、统一判定口径

| 规则项 | 统一口径 |
|------|------|
| 配置拆分 | 技能强化件配置拆为两部分：`skillEnhancementItemConfig` 负责属性，`skillEnhancementUpgradeConfig` 负责品质与升级消耗 |
| 属性读取条件 | 当前等级属性只按 `itemId + level` 读取，不额外拼 `quality` |
| 升级消耗读取条件 | 当前等级升级消耗按 `itemId + level` 从 `upgradeCosts` 中定位；`quality` 仅作描述信息 |
| 品质来源 | 强化件品质统一从 `skillEnhancementUpgradeConfig.quality` 带出，不在属性对象内重复配置 |
| 背包存储模型 | 技能强化件统一按背包 `stackItemRecord` 存储，当前等级写入 `level` 字段 |
| 堆叠聚合条件 | 同一玩家背包下，同一 `itemId`、同一 `level`、同一过期批次的技能强化件按一条堆叠记录聚合 |
| 默认等级 | 新获得且未升级的技能强化件默认 `level=1` |
| 升级消耗物品 | 升级时只消耗 `fragmentItemId` 对应的强化件碎片，不消耗强化件本体数量 |
| 升级结算对象 | 每次升级针对单个强化件本体生效；若该强化件在堆叠记录中存在多件，只移动其中1件到新等级堆叠记录 |
| 升级成功后的库存变化 | 升级成功后，原等级 `stackItemRecord.count - 1`；目标等级 `stackItemRecord.count + 1`，若目标等级记录不存在则新建 |
| 满级限制 | 当前等级已达到配置上限时，不再返回下一等级升级消耗，也不允许继续升级 |
| 属性生效口径 | 属性配置中的数值均为该等级最终值，不做“上一等级 + 增量”二次计算 |
| 零值字段处理 | 某强化件不涉及的属性字段固定填 `0`，运行时按“不生效”处理 |
| 读取详情结果 | 对外返回技能强化件详情时，至少包含：背包堆叠对象、当前等级属性、当前等级升级信息、品质 |
| 升级事务 | 升级需按“校验强化件存在 → 校验未满级 → 校验碎片足够 → 扣碎片 → 迁移强化件等级记录”原子提交；任一步失败整笔回滚 |
| 并发升级限制 | 同一件强化件同一时刻只允许1次升级成功，避免重复扣碎片或等级跳变 |

## 六、核心场景用例（仅逻辑与预期数据）

| 用例ID | 场景 | 前置数据 | 触发 | 预期数据 |
|------|------|----------|------|----------|
| C01 | 新获得强化件入背包 | 背包中不存在 `itemId=1001, level=1` 的强化件记录 | 发放强化件 `1001 x1` | 新增一条 `stackItemRecord(itemId=1001, level=1, count=1)` |
| C02 | 已有同等级强化件继续叠加 | 已存在 `itemId=1001, level=1, count=2` | 再获得强化件 `1001 x3` | 同一条记录聚合为 `count=5` |
| C03 | 读取1级强化件详情 | 背包存在 `itemId=1001, level=1, count=2`；属性配置存在 `(1001,1)`；升级配置存在 `itemId=1001` 且含 `level=1` 消耗 | 请求读取强化件详情 | 返回背包记录、1级属性、1级升2级消耗、以及 `quality` |
| C04 | 单件强化件从1级升到2级 | 背包存在 `itemId=1001, level=1, count=3`；碎片足够；存在 `level=1` 升级消耗 | 执行1次升级 | 扣除对应碎片；`level=1` 记录减为 `count=2`；`level=2` 记录加1或新建；返回2级属性 |
| C05 | 目标等级记录已存在时合并 | 背包已有 `itemId=1001, level=1, count=1` 与 `itemId=1001, level=2, count=4` | 执行1次1级升2级 | 升级后 `level=1` 记录清空或删除；`level=2` 记录变为 `count=5` |
| C06 | 不同等级强化件分开存储 | 背包同时存在 `itemId=1001, level=1` 与 `itemId=1001, level=3` | 读取背包列表 | 两条记录独立展示，不允许跨等级聚合 |
| C07 | 按当前等级读取最终属性 | 背包存在 `itemId=2001, level=5, count=1`；5级属性配置已填最终值 | 请求读取详情 | 直接返回5级最终属性，不再叠加1~4级配置 |
| C08 | 品质从升级配置带出 | `itemId=2001` 的升级配置顶层 `quality=3`；属性配置不含 `quality` | 请求读取详情 | 返回的品质为 `3`，属性配置结构不变 |
| C09 | 升级消耗随等级变化 | `itemId=3001` 的 `upgradeCosts` 已配置1级、10级、20级对应消耗 | 分别读取1级、10级、20级升级信息 | 返回各自等级对应的碎片ID与数量，不混用其他等级消耗 |
| C10 | 满级强化件读取详情 | 背包存在 `itemId=1001, level=30, count=1`；属性配置存在30级数据 | 请求读取详情 | 返回30级属性；升级信息为空或标记满级不可升级 |
| C11 | 多件同等级强化件连续升级 | 背包存在 `itemId=1001, level=1, count=5`；碎片充足 | 连续执行2次升级 | 最终 `level=1` 减少2件，`level=2` 增加2件；每次都只处理1件本体 |
| C12 | 零值属性字段正常返回 | 某强化件仅配置 `attackAdd`，其他字段均为 `0` | 请求读取详情 | 返回完整属性对象；未生效字段仍保留且值为 `0` |

## 七、边界情况用例（仅逻辑与预期数据）

| 用例ID | 场景 | 前置数据 | 触发 | 预期数据 |
|------|------|----------|------|----------|
| B01 | 背包中不存在强化件本体 | 背包无目标 `itemId` 记录 | 请求读取或升级该强化件 | 返回“物品不存在”；不扣碎片、不生成等级记录 |
| B02 | 属性配置缺失当前等级 | 背包存在 `itemId=1001, level=6`；缺少 `(1001,6)` 属性配置 | 请求读取详情 | 返回配置错误；不得用其他等级属性兜底 |
| B03 | 升级配置缺失当前等级消耗 | 背包存在 `itemId=1001, level=6`；升级配置中无 `level=6` 项 | 请求升级 | 返回配置错误；不扣碎片、不改库存 |
| B04 | 碎片不足时升级失败 | 强化件存在；目标等级消耗为 `costItemCount=5`；玩家仅有4个碎片 | 执行升级 | 返回“碎片不足”；强化件等级与碎片库存保持不变 |
| B05 | 满级后重复升级 | 背包存在 `itemId=1001, level=30, count=1` | 执行升级 | 返回“已满级”；不扣碎片、不生成31级记录 |
| B06 | 同一件强化件并发升级 | 同一条 `itemId=1001, level=1, count=1` 记录；碎片只够1次 | 两个请求同时升级 | 仅1次成功；碎片只扣1次；最终只生成1件2级强化件 |
| B07 | 原等级记录减到0 | 背包存在 `itemId=1001, level=1, count=1`；升级成功 | 执行升级 | 原 `level=1` 记录删除或清理为0后移除；仅保留目标等级记录 |
| B08 | 不同等级记录不能误合并 | 背包存在 `itemId=1001, level=1, count=2` 与 `itemId=1001, level=2, count=2` | 发放1件 `itemId=1001, level=1` | 只增加 `level=1` 记录；不得并入 `level=2` |
| B09 | 读取品质时属性配置无品质字段 | 属性配置结构无 `quality`；升级配置存在 `quality=4` | 请求读取详情 | 仍能正确返回品质 `4`；不得因属性配置无品质而报错 |
| B10 | 等级字段异常 | 背包记录 `level=0` 或 `level` 超出配置上限 | 请求读取或升级 | 返回非法等级错误；不得按默认1级自动修正 |
| B11 | 升级扣碎片成功但迁移等级失败 | 扣碎片后写入目标等级记录失败 | 提交升级事务 | 整笔升级回滚；碎片恢复；原等级记录不变 |
| B12 | 同一 `itemId` 不同过期批次共存 | 同一 `itemId`、同一 `level` 但 `expireAt` 不同 | 读取或升级其中一批 | 仅处理目标批次记录；不得错误影响另一批 |
